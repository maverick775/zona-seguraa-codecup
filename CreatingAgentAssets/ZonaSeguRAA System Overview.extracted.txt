1. Visión del Producto
Zona SeguRAA es un ecosistema de seguridad comunitaria inteligente que combina una red de dispositivos IoT LoRaWAN con una plataforma web de gestión de alertas en tiempo real. Para el Hackathon CodeCup 2026, construimos el MVP funcional demostrando el flujo completo de respuesta a emergencias en el Estadio Akron durante el Mundial de Fútbol 2026.
Propuesta de valor central del MVP:Un visitante extranjero activa una alerta de emergencia médica desde la app en su idioma → la comunidad presente valida el reporte en tiempo real con comentarios contextuales → el coordinador de seguridad gestiona la respuesta desde el dashboard → el dispositivo físico AVP3 activa señales luminosas/sonoras en la zona. Todo el flujo es medible en segundos.
2. Escenario de Demostración
Historia: Dos turistas extranjeros asisten al Estadio Akron. Uno pierde el conocimiento en las gradas. Su acompañante abre la app, selecciona idioma inglés, activa el botón de pánico con tipo "Medical Emergency", e ingresa su ubicación (fila y asiento).
El reporte aparece en el dashboard traducido al español. Otros espectadores cercanos notan la alerta activa en la app y validan votando, agregando comentarios ("Row 5, Seat 14, person is unconscious"). Al acumularse 3 votos, la alerta escala automáticamente a Nivel 2. El coordinador de seguridad asigna el estatus EN ATENCIÓN desde el dashboard, lo cual activa señales en el nodo físico AVP3 de la zona correspondiente.
KPI visible durante la demo:
⏱️ Tiempo de respuesta: 42 segundos desde activación hasta EN ATENCIÓN

5. Modelo de Datos
5.1 Tablas principales
-- Zonas de cobertura
zones (
id uuid PRIMARY KEY,
name text, -- "Estadio Akron - Zona A"
description text,
city text, -- "Guadalajara"
created_at timestamp
)
-- Nodos IoT (físicos y simulados)
nodes (
id uuid PRIMARY KEY,
zone_id uuid REFERENCES zones,
name text, -- "AVP3-Alfa-01"
type text, -- 'avp_physical' | 'avp_simulated' | 'medical_point' | 'exit'
x float, -- posición en mapa estático (% del ancho)
y float, -- posición en mapa estático (% del alto)
lorawan_eui text, -- DevEUI si es nodo físico, null si simulado
status text, -- 'online' | 'offline' | 'alert'
last_seen_at timestamp
)
-- Alertas
alerts (
id uuid PRIMARY KEY,
zone_id uuid REFERENCES zones,
node_id uuid REFERENCES nodes,
type text, -- 'medical' | 'security' | 'evacuation'
level int, -- 1 | 2 | 3
status text, -- 'active' | 'in_progress' | 'resolved' | 'false_alarm'
description text,
language text, -- idioma del reporte original ('en' | 'es')
created_by_nick text,
created_at timestamp,
updated_at timestamp,
resolved_at timestamp,
attended_by uuid REFERENCES users_coord
)
-- Votos y comentarios de la comunidad
alert_votes (
id uuid PRIMARY KEY,
alert_id uuid REFERENCES alerts,
nickname text,
role text, -- 'visitor' | 'coordinator'
weight int, -- visitor=1, coordinator=2
comment text,
created_at timestamp
)
-- Usuarios temporales (visitantes)
users_temp (
id uuid PRIMARY KEY,
nickname text UNIQUE,
zone_id uuid REFERENCES zones,
language text,
created_at timestamp,
expires_at timestamp -- now() + interval '1 hour'
)
-- Coordinadores de seguridad
users_coord (
id uuid PRIMARY KEY,
email text UNIQUE,
role text DEFAULT 'coordinator',
created_at timestamp
-- password gestionado por Supabase Auth
)
-- Log de uplinks IoT (para debug y trazabilidad)
iot_uplinks (
id uuid PRIMARY KEY,
node_eui text,
payload_hex text,
alert_state int,
battery_pct int,
received_at timestamp
)
5.2 Reglas de negocio
Escalada automática a Nivel 2: SUM(alert_votes.weight) >= 3 sobre una alerta activa dispara level = 2.
Nivel 3: Solo coordinadores pueden escalar; activa notificación de evacuación en nodos.
Expiración de usuarios temporales: Job de Supabase (pg_cron) o limpieza lazy al registrar nuevos.
Downlink al AVP: Al cambiar alerts.status = 'in_progress', el backend publica downlink vía API de Loriot al node.lorawan_eui correspondiente.

9. Plan de Contingencia (Wizard of Oz)
Si el hardware IoT falla durante la presentación, el sistema opera en Modo Demo:
Script scripts/demo-mode.js ejecutable manualmente.
Genera la secuencia completa de eventos vía llamadas directas a /api/alerts y /api/votes, simulando el flujo exacto del escenario del extranjero.
El dashboard muestra exactamente el mismo comportamiento visual que con el hardware real.
Los dispositivos físicos quedan como props visuales (ya tienen el firmware cargado y pueden mostrar estados en OLED).
Activación: node scripts/demo-mode.js --zone akron --scenario medical
10. Definition of Done (DoD) del MVP
Los siguientes criterios, si se cumplen, garantizan una demo exitosa independientemente de todo lo demás:
DoD-1: URL pública zonaseguraa.com/zone/akron carga el mapa del Estadio Akron con nodos marcados en <3 segundos.
DoD-2: Un visitante puede registrarse con alias, activar una alerta de emergencia médica y ver confirmación en <10 segundos.
DoD-3: El estado de la alerta cambia visiblemente en el dashboard del coordinador en tiempo real al recibir votos de la comunidad.
DoD-4: El coordinador puede cambiar el estado de una alerta a EN ATENCIÓN y el cambio se refleja en el dashboard sin recargar la página.
DoD-5: El historial de eventos registra cada transición con timestamp correcto.
DoD-6 (opcional): Un uplink del AVP físico actualiza el estado del nodo correspondiente en el mapa en <15 segundos.
